このシステムは、「面倒くさい」「危ない」「忘れる」を完全に排除するために設計された、自動化された運用システム（エコシステム）です。

3.  **「セットアップが面倒」をゼロにする** (CLI自動化)

## 💻 3. VS Code プロファイル管理の仕組み

### 🚫 禁止事項 (セーフティー制御)
* **設定画面でフォントサイズなどを変更すること。** (ロックされています)
* **歯車アイコンからプロファイルを新規作成・削除すること。** (管理外警告が出ます)

### ✅ 正しい変更フロー
1.  `dev` から **「🔓 Unlock Settings」** を選ぶ。
2.  GUIで自由に設定を変えて試す。
3.  **「⚙️ Apply & Lock」** を選んで、変更を確定（ロック）する。

これにより、**「うっかり設定を変えてしまって、元に戻せない」という事故が物理的に起きなくなります。**

## 📂 4. フォルダ管理の概念 (仮想PARA)

* **裏側（倉庫）**: `~/Projects` (コード) と `~/Creative` (素材) に分離。
* **表側（デスク）**: **`~/PARA/1_Projects`** が入り口。ここには、上記の「コード」と「素材」へのショートカットが自動で置かれています。

## 🔐 5. セキュリティ

* **パスワード管理:** `Bitwarden` と連携し、APIキーなどはコマンド一つで環境変数に注入されます。
* **漏洩防止:** コミット時・プッシュ時に、秘密情報が含まれていないか自動チェックが走ります (`pre-commit`)。
* **脆弱性診断:** Snyk (VS Code) と Trivy (CLI) で、コードを書いている最中から安全性をチェックします。
